# プロジェクト指示: mobile-app-poc-mock

## プロジェクト概要
OpenAPI 契約 (`openapi.yaml`) から **モックAPIサーバー** を立てるリポジトリ。
Prism (Stoplight) を使用。バックエンド未実装でもフロント開発を進められる状態を提供する。

## 位置づけ
モバイルアプリ開発環境調査プロジェクトの1リポジトリ。以下5リポジトリのうちの1つ:
- mobile-app-poc-api-spec（ここから openapi.yaml を取得）
- **mobile-app-poc-mock** ← このプロジェクト
- mobile-app-poc-frontend（このモックを参照）
- mobile-app-poc-bridge
- mobile-app-poc-backend

## 技術スタック
- Prism CLI (`@stoplight/prism-cli`)
- OpenAPI 3.0.3
- Node.js 18+（LTS 22.x 推奨）

## 開発環境・公開範囲
- OS: Windows ネイティブ（WSL2 / Docker は使わない）
- **グローバル非依存**: `npm install` でローカル `node_modules` に展開、`npm run mock` で起動
- ポート: 4010（競合時は別ポートへ）
- **公開範囲は社内イントラネット内のみ**。外部公開（Tailscale Funnel / ngrok / クラウドデプロイ等）は対象外。

## 起動コマンド
```powershell
npm install
npm run mock           # http://localhost:4010
npm run mock:dynamic   # 動的モック（faker）
```

## 社内共有方法の選択肢
1. **ローカル起動のみ**（各開発者が自分のPCで `npm run mock`）
   → 最もシンプル。調査フェーズの基本形
2. **LAN 内の共有PC/サーバーで起動**（同一セグメントからアクセス可）
   → `--host 0.0.0.0` で起動、ファイアウォール 4010 許可
3. **Windows サービス化（nssm 等）**
   → 常時稼働が必要な場合
```powershell
nssm install PrismMock "C:\Program Files\nodejs\node.exe" `
  "C:\work\mobile-app-poc-mock\node_modules\@stoplight\prism-cli\dist\index.js mock openapi.yaml --host 0.0.0.0"
nssm start PrismMock
```

## Windows 固有の注意点
- PowerShell 実行ポリシー: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`
- ファイアウォール: 4010 ポート受信許可（LAN共有時のみ）
- ポート競合: Hyper-V 予約範囲 (`netsh interface ipv4 show excludedportrange`) 確認
- 改行コード: `.gitattributes` で `*.yaml *.json text eol=lf`
- 社内プロキシ環境下では `npm config set proxy ...` 必要な場合あり

## JSON 参照パターン
- example: `externalValue: './examples/xxx.json'`
- schema: `$ref: './schemas/xxx.json'`
- 複数パターン切替: `Prefer: example=xxx` ヘッダ
