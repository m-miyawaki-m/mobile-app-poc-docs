# プロジェクト指示: mobile-app-poc-frontend

## プロジェクト概要
モバイルアプリのフロントエンド実装。Ionic + Vue + Capacitor を採用し、
Android 実機で動作することを目標とする。API は別サーバーから取得する構成
（開発時: Prism モック、本番: Spring Boot）。

## 位置づけ
モバイルアプリ開発環境調査プロジェクトの1リポジトリ。以下5リポジトリのうちの1つ:
- mobile-app-poc-api-spec（ここから openapi.yaml を取得し TS 型生成）
- mobile-app-poc-mock（開発時の API 接続先）
- **mobile-app-poc-frontend** ← このプロジェクト
- mobile-app-poc-bridge（aar 連携プラグインをインストール）
- mobile-app-poc-backend（将来の本番 API 接続先）

## 技術スタック
- Ionic 8.x
- Vue 3（Composition API）
- TypeScript
- Vite
- Capacitor 6.x
- Pinia（状態管理）
- Ionic Router
- axios or CapacitorHttp
- OpenAPI 型生成: `openapi-typescript` or `orval`

## 開発環境
- OS: Windows ネイティブ
- Node.js 18+（LTS 22.x 推奨）
- Android Studio（実機デバッグ用）
- エディタ: VS Code + Volar
- 接続先APIは社内LAN内のみ

## ディレクトリ構成
```
mobile-app-poc-frontend/
├─ src/
│  ├─ api/
│  │  ├─ generated/       # openapi-typescript 出力
│  │  └─ client.ts        # CapacitorHttp ラッパー
│  ├─ views/
│  ├─ components/
│  ├─ stores/             # Pinia
│  ├─ router/
│  └─ main.ts
├─ android/               # npx cap add android 後
├─ .env.development       # VITE_API_BASE=http://localhost:4010
├─ .env.device            # 実機用（PC の LAN IP）
├─ .env.production
├─ capacitor.config.ts
├─ ionic.config.json
└─ package.json
```

## API クライアント方針
- `CapacitorHttp` プラグイン採用（CORS 影響を受けない）
- `openapi-typescript` で TS 型自動生成
- 環境変数 `VITE_API_BASE` でエンドポイント切替

## 主要コマンド
```powershell
npm install
ionic serve                      # ブラウザで開発
npx cap sync android             # ネイティブ同期
npx cap open android             # Android Studio 起動
npx cap run android --livereload # 実機 + Live Reload
```

## 調査観点
- Ionic Vue コンポーネントの使用感
- Vite + TS + Vue 3 の DX
- Pinia の状態管理パターン
- Ionic Router と Vue Router の違い
- `CapacitorHttp` vs `axios` の使い分け
- OpenAPI 型生成の運用負荷
- iOS/Android 両対応のコスト
- Hot Reload の実機動作

## 回答方針
- 詳細な回答の前に概要を回答すること
- Composition API (`<script setup>`) で記述
- Ionic Vue 固有の記法（Option API の Vue とは違う）に注意
- Windows PowerShell 前提のコマンドで提示
- サンプルデータは汎用的な値を使用
