# プロジェクト指示: mobile-app-poc-api-spec

## プロジェクト概要
モバイルアプリとバックエンドの間の **API 契約** を Single Source of Truth として管理するリポジトリ。
OpenAPI 3.0.3 形式の `openapi.yaml` を中心に、スキーマ定義 (JSON) と
レスポンス例 (JSON) を分離管理する。

## 位置づけ
モバイルアプリ開発環境調査プロジェクトの1リポジトリ。以下5リポジトリのうちの1つ:
- **mobile-app-poc-api-spec** ← このプロジェクト
- mobile-app-poc-mock（この YAML を参照して Prism で起動）
- mobile-app-poc-frontend（この YAML から TypeScript 型を自動生成）
- mobile-app-poc-bridge
- mobile-app-poc-backend（この YAML から Controller 雛形を生成）

## 技術スタック
- OpenAPI 3.0.3
- Stoplight Spectral（lint）
- （任意）Redocly / Swagger UI（ドキュメント表示）

## 開発環境
- OS: Windows ネイティブ
- Node.js 18+（lint ツール実行のため）
- パッケージ管理はプロジェクトローカル
- 社内イントラネット内での運用のみ

## ディレクトリ構成
```
mobile-app-poc-api-spec/
├─ openapi.yaml           # メイン定義
├─ schemas/               # 再利用スキーマ（JSON）
├─ examples/              # レスポンス例（JSON）
│  └─ errors/
├─ .spectral.yaml         # lint 設定
├─ package.json
└─ README.md
```

## 設計方針
- スキーマ定義は `schemas/*.json` に切り出し `$ref` で参照
- レスポンス例は `examples/*.json` に切り出し `externalValue` で参照
- 複数パターンの example を用意（正常/空/エラーなど）
- lint を通さないと merge できない運用を想定

## 回答方針
- 詳細な回答の前に概要を回答すること
- OpenAPI 記法で迷ったら 3.0.3 仕様に準拠
- エンドポイント追加時は schema / example / error も一緒に提示
- 破壊的変更は明示的に警告
- サンプルデータは汎用的な値（ユーザー、商品、在庫、タスク等）を使用
