# プロジェクト指示: mobile-app-poc-backend

## プロジェクト概要
モバイルアプリ向けのバックエンド API を Spring Boot で実装する。
OpenAPI 契約 (`openapi.yaml`) から **Schema-First** で Controller 雛形を自動生成する方針。

## 位置づけ
モバイルアプリ開発環境調査プロジェクトの1リポジトリ。以下5リポジトリのうちの1つ:
- mobile-app-poc-api-spec（ここから openapi.yaml を取得し Controller 生成）
- mobile-app-poc-mock
- mobile-app-poc-frontend（このAPIを将来的に参照）
- mobile-app-poc-bridge
- **mobile-app-poc-backend** ← このプロジェクト

## 技術スタック
- Spring Boot 3.x
- Java 21（LTS）
- Gradle
- MyBatis（既存 SES 案件との親和性）
- Oracle DB（本番想定、開発は H2 で代替可）
- Spring Security + JWT
- springdoc-openapi（ドキュメント）
- openapi-generator-gradle-plugin（Schema-First コード生成）

## 開発環境・公開範囲
- OS: Windows ネイティブ
- JDK 21
- エディタ: VS Code or IntelliJ IDEA
- ビルド: Gradle Wrapper
- **公開範囲は社内イントラネット内のみ**（外部公開は対象外）

## ディレクトリ構成
```
mobile-app-poc-backend/
├─ src/main/java/com/example/app/
│  ├─ Application.java
│  ├─ config/
│  │  ├─ CorsConfig.java
│  │  └─ SecurityConfig.java
│  ├─ controller/          # 生成されたインターフェースを実装
│  ├─ service/
│  ├─ mapper/              # MyBatis
│  └─ domain/
├─ src/main/resources/
│  ├─ application.yml
│  ├─ mapper/              # MyBatis XML
│  └─ openapi/
│     └─ openapi.yaml      # api-spec からコピー
├─ build/generated/        # openapi-generator 出力
├─ build.gradle
├─ settings.gradle
└─ README.md
```

## CORS 設定（Capacitor 対応）
許可オリジン:
- `http://localhost:8100`（ionic serve）
- `capacitor://localhost`（iOS）
- `http://localhost`（Android）

## 認証方針
- JWT ベース
- `/auth/login` → JWT 発行
- `Authorization: Bearer <token>` で検証
- Spring Security で `/auth/**` と Swagger UI 以外は要認証

## 主要コマンド
```powershell
.\gradlew bootRun                 # 起動
.\gradlew build                   # ビルド
.\gradlew openApiGenerate         # Controller 雛形生成
.\gradlew test                    # テスト
```

## Schema-First フロー
1. `src/main/resources/openapi/openapi.yaml` を配置（api-spec から）
2. `openApiGenerate` タスクで Controller インターフェースを生成
3. 実装クラスで `implements XxxApi` し、メソッドを実装
4. YAML 更新 → 再生成 → 実装反映

## 調査観点
- Spring Boot 3.x + Java 21 の Windows ネイティブ構築
- Schema-First のコード生成運用負荷
- MyBatis + Oracle の接続方法
- JWT 認証の実装パターン
- Capacitor 向け CORS 設定
- WebLogic との互換性（本番デプロイ想定、社内環境内）
- Gradle / Maven どちらが扱いやすいか

## 回答方針
- 詳細な回答の前に概要を回答すること
- Java 21 の記法（record, var, switch expression, pattern matching）を積極活用
- Spring Boot 3.x 流儀（`jakarta.*` パッケージ）
- 既存 SES 資産（Struts2/WebLogic/MyBatis/Oracle）との互換性に配慮
- Windows PowerShell 前提
- `.\gradlew` 形式でコマンドを提示
- サンプルデータは汎用的な値を使用
- 外部公開向けの設定（インターネット配信・CDN 等）は提案しない
